DATABASE STRATEGY (cost-aware)
- RDS (authoritative relational core): users, auth, topics, characters, posts, comments, series metadata.
- DynamoDB (high-write/event workloads): reactions stream + counters, notifications inbox, messages timeline, voice votes/takes event fanout.
- Start with RDS for correctness and faster delivery; add DynamoDB in phase 2+ where write load and cost benefit are clear.
